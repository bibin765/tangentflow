Security Practices

Overview

TangentFlow is designed with security-first principles. The most important thing to know: we do not store your document data. Documents are generated in-memory, returned to you, and immediately discarded. There is nothing to breach because there is nothing to store.

Architecture

Documents are generated in-memory within isolated processes. PDF bytes are streamed directly back to the client and never written to disk. Each request is handled independently with no shared state between requests.

Encryption

All API traffic is encrypted via TLS 1.3. API keys are generated using cryptographically secure random bytes, ensuring they cannot be predicted or brute-forced.

Authentication

API access is authenticated via Bearer token API keys included in request headers. Keys can be revoked at any time through the dashboard. Revoked keys are rejected immediately — there is no grace period.

Infrastructure

TangentFlow is hosted on Hetzner Cloud in Germany, within the European Union. The application runs in Docker containers with a minimal attack surface — no unnecessary packages, no shell access, no root processes.

No Data Retention

Document schemas sent to the API and the generated PDF output are processed entirely in-memory. Nothing is logged, cached, or written to persistent storage. Once the response is sent, the data is gone.

Rate Limiting

Per-key rate limits are enforced to prevent abuse and ensure fair usage across all customers. Rate limits are configurable per pricing tier, and requests that exceed the limit receive a clear error response.

Open Source

The core library (@upbrew/tangentflow) is fully open source and available on GitHub. You can audit the code, review the algorithms, and verify that the library does exactly what it claims — nothing more.

npm Provenance

Published npm packages are cryptographically linked to their GitHub source via npm provenance. This means you can verify that any published version was built directly from the public repository, with no tampering in between.

Vulnerability Reporting

If you discover a security vulnerability, please report it responsibly to security@tangentflow.com. For details on our disclosure process, see SECURITY.md in the GitHub repository.

Compliance

TangentFlow is GDPR-compliant by design. With no document data retention and all servers located in the EU, there is minimal data exposure by architecture rather than by policy alone.

  • GDPR — compliant (no data retention, EU servers, minimal data collection)
