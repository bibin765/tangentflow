We believe PDF generation should be simple, accurate, and open.

Every developer has the same experience with PDF generation. You pick a library, write the code, and everything looks great — until you add a table with long text, or switch to a different language, or test on mobile. Then the text overflows, the columns break, and you spend three days debugging something that should just work.

The root cause is fundamental: PDF is a fixed-layout format. Unlike HTML, there is no layout engine to reflow text. Every PDF library must implement its own text measurement and line-breaking algorithm. Most take shortcuts — approximate character widths, ASCII-only assumptions, or requiring a headless browser as a crutch.

TangentFlow was built on a different foundation. We use Pretext, a JavaScript library that measures text with mathematical precision using the browser's actual font engine. No approximations, no heuristics, no headless Chrome. Just accurate layout, in pure arithmetic, for every language and every script.

Text Layout Engine

Pretext

A JavaScript library for precise text measurement and paragraph layout. Created by Cheng Lou, Pretext solves the hardest problem in programmatic PDF generation — knowing exactly where each glyph should go. It handles Latin, CJK, Arabic, Hindi, emoji, and mixed-direction text with full Unicode support.

PDF Generation

pdf-lib

A JavaScript library for creating and modifying PDF documents in any JavaScript environment. pdf-lib handles the PDF file format — pages, fonts, drawing commands, and binary encoding — so TangentFlow can focus entirely on layout and design.

What we care about.

Accuracy first

Every pixel matters in document generation. A line break in the wrong place can change the meaning of a sentence, split a table awkwardly, or make a professional document look broken. We don't approximate — we measure.

No black boxes

Open source, no hidden servers, no data collection. Your documents are generated in your browser and never leave your machine. The code is readable, the algorithms are documented, and the results are reproducible.

Developer-friendly

Built by developers, for developers. Clean APIs, sensible defaults, and progressive complexity. Start with the visual builder, graduate to the JSON schema, scale to the batch API — at your own pace.
